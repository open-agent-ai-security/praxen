BEHAVIOR SUMMARY
------------------------------------------------------------------------------
The dominant pattern is prompt-as-the-only-layer: every security-relevant
behavioral constraint the remit demands — no system-prompt disclosure, no
following of instructions embedded in retrieved content, no acceptance of
fabricated history — lives exclusively in the `system.instructions` block of
`haaHelpAgent.agent`, with no code-level interposition on user input, tool
output, or agent responses in either UI path. The most exploitable consequence
is indirect prompt injection via Knowledge articles:
`AnswerQuestionsWithKnowledge` returns article rich text with
`filter_from_agent: False`, so a poisoned article's instructions enter the LLM
context as trusted tool output and bypass all prompt-level guardrails — and
because no durable, action-level logging is wired by default and the remit's
halt/alert escalation rules have no implementation behind them, such an attack
would execute and leave no trace.

CRITICAL FINDINGS
------------------------------------------------------------------------------
  PRAX-2026-07-10-001  Retrieved Knowledge article content enters the LLM
            context unfiltered and unmonitored — indirect prompt injection is
            exploitable and leaves no trace
      Action 1: Add a system-level instruction that explicitly labels
                `AnswerQuestionsWithKnowledge` output as untrusted external
                content that must never be treated as operator instructions,
                and evaluate enabling a post-retrieval sanitization step (or
                `filter_from_agent: True` where the platform supports it)
                before article content reaches the LLM context.
      Action 2: Make Einstein Audit and Feedback and Agentforce Session
                Tracing mandatory deployment prerequisites so that a routing
                or tool-invocation change caused by poisoned article content
                is recorded and alertable.

HIGH FINDINGS
------------------------------------------------------------------------------
  PRAX-2026-07-10-002  All disclosure and fabricated-history controls are
            prompt-only — no code-level output filter or halt mechanism exists
            in either UI path
      Action 1: Add a post-response output filter (at the platform action
                level or in the Embedded Messaging event handler) that scans
                agent responses for system-prompt-shaped or
                configuration-value content before display and alerts the
                operator on a match.
      Action 2: Strengthen the system prompt with an explicit false-history
                clause rejecting claims of permissions not established in the
                current session's system turn, as an interim measure while a
                code-level filter is built.

  PRAX-2026-07-10-003  The off_topic topic instructs the agent to offer
            human-agent escalation that the remit prohibits and no routing
            path supports
      Action: Remove the "ask whether they want to escalate to a human agent"
              clause from the `off_topic` topic instructions and replace it
              with the same website-support redirect language used in the
              `escalation` topic.

  PRAX-2026-07-10-004  No durable action-level logging is wired by default —
            tool calls, topic routing, and session events are unrecorded
      Action 1: Reclassify Einstein Audit and Feedback and Agentforce Session
                Tracing as required deployment prerequisites in the README and
                setup flow, alongside the Data Library and Embedded Service
                Deployment steps.
      Action 2: Add a go-live checklist item that verifies audit logging is
                active before the agent is activated.

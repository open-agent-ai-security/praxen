BEHAVIOR SUMMARY
------------------------------------------------------------------------------
The sandbox the remit relies on is not in the code. src/sandbox/code_runner.py
and src/sandbox/firejail.py are both zero-line stubs, while the real execution
path — src/agents/runner/runner.py run_code() — calls
subprocess.run(command_set) on whatever shell strings the LLM returns, with no
allowlist, no per-command approval, and a 2-attempt retry loop that re-prompts
the LLM on failure. This sandbox-shaped-but-empty pattern compounds with three
other gaps to form a one-hop external-to-shell chain: an unauthenticated POST
/api/settings route in devika.py that rewrites API keys and the OpenAI base
URL (allowing model-endpoint hijack), the Researcher fetching web pages via
Bing/Google/DuckDuckGo and feeding their raw content through Formatter into
the next LLM turn with zero injection filtering, and
Coder/Patcher/Feature.save_code_to_project() writing LLM-supplied filenames
joined to project_dir with no traversal check. Together: an attacker who can
either reach 0.0.0.0:1337 or seed a top web result can redirect the model,
write files outside the project dir, and execute arbitrary commands — with no
per-project audit log to detect any of it.

  PRAX-2026-05-11-001  Sandbox is a 0-line stub; Runner executes LLM-supplied

              not retry an unapproved command
      Action 3: Disable the rerunner retry loop (src/agents/runner/runner.py

              session

              agent if exec is enabled

  PRAX-2026-05-11-002  Coder/Patcher/Feature.save_code_to_project() writes

      Action 1: Add a shared safe_join(project_root, llm_path) helper that

      Action 3: Add a unit test that feeds /etc/passwd, ../foo, and a

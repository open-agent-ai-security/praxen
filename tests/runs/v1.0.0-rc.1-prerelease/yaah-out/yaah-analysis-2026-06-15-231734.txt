BEHAVIOR SUMMARY
------------------------------------------------------------------------------
The dominant pattern is policy-implementation divergence concentrated at one
seam: the harness's central promise of uniform protective hooks across every
generated agent is broken for the Codex CLI target. `pkg/generator/hookmap.go`
maps no Codex event name for PreToolUse or PostToolUse, so
`CodexGenerator.GenerateHooks` skips them and ships a `.codex/hooks.json` with
only SessionStart and Stop — none of the advertised command-guard,
secret-scanner, linter, or comment-checker — and a passing test
(`TestCodex_GenerateHooks_NoSupported`) encodes the gap as intended. The
operative controls themselves are real and run on the agent's path: a regex
command-guard that blocks catastrophic shell commands, a secret scanner that
blocks Edit/Write of files carrying credentials, and a durable structured
session audit log. Secondary themes are supply-chain (the `context7` MCP
installed via unpinned `npx -y @context7/mcp`), a narrow command-guard
denylist, an MCP write tool (`yaah_planning_init`) with no approval gate, and
absent adversarial testing / security-disclosure docs.

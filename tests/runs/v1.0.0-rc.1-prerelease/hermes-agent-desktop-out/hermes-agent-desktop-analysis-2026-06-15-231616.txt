BEHAVIOR SUMMARY
------------------------------------------------------------------------------
This is a high-maturity, actively-maintained single-tenant agent whose
security model is honest about its own limits: its own SECURITY.md states
plainly that the only boundary against an adversarial LLM is the operating
system, and that the approval gate, output redaction, Skills Guard, and tool
allowlists are heuristics, not containment. The dominant risk is the gap
between that stated model and the default deployment: the local terminal
backend runs LLM-emitted shell and file operations directly on the host, while
untrusted content (open web, inbound email, multi-user gateways,
operator-configured MCP servers) flows into the same agent context — so a
single indirect prompt injection reaches host execution unless the operator
has separately stood up Docker/OpenShell whole-process isolation that the code
does not require by default. Layered on that are second-order gaps the team
has largely disclosed: dangerous-shell auto-approval fails open in
non-interactive/non-gateway/non-cron execution, messaging adapters default to
open/pair when no caller allowlist is set, MCP servers launch and their tools
execute with no operator-review or approval gate, and the Desktop ships
default-on telemetry with no non-root SSH guard and trust-on-first-use host
keys. These sit against genuinely strong, operative controls — a fail-closed
API-server adapter, an un-overridable credential blocklist (the
GHSA-rhgp-j443-p4rf fix), a hardline command floor below YOLO mode,
exact-pinned deps with supply-chain CI, and redacting rotating logs — which is
why the posture lands at the Established/Partial boundary rather than lower.

CRITICAL FINDINGS
------------------------------------------------------------------------------
  PRAX-2026-06-15-001  Default `local` terminal backend executes LLM-emitted
            shell on the host while untrusted web/email/MCP content is
            ingested with only in-process heuristics in force.
      Action 1: Default to a non-`local` terminal backend (or whole-process
                Docker/OpenShell wrapping) whenever any untrusted input
                surface (gateway, email, web, MCP) is enabled, and refuse to
                ingest those surfaces under the `local` backend without an
                explicit operator break-glass acknowledgement.
      Action 2: Surface the chosen isolation posture in the runtime
                banner/logs so an operator running `local` with untrusted
                surfaces is warned at startup.

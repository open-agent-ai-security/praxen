------------------------------------------------------------------------------
The dominant pattern is a sandbox with the shape of isolation but none of the
substance, feeding a single catastrophic compound chain.
`src/sandbox/firejail.py` and `src/sandbox/code_runner.py` — the only files
whose names promise execution isolation — are 0-byte stubs, while
`ARCHITECTURE.md` claims code is "executed in a sandboxed environment." In
their absence, untrusted web content fetched by
`researcher`/`DuckDuckGoSearch` and the user prompt flow unscreened through
`formatter` into `coder` and `runner`, where `Runner.run_code()` passes
LLM-produced command strings straight to `subprocess.run()` on the host with
no approval gate, and `Coder.save_code_to_project()` joins an LLM-controlled
filename with no traversal guard. An unauthenticated `/api/settings` POST and
arbitrary-file-read via `/api/get-browser-snapshot` sit on the same
`0.0.0.0:1337` bind, and there are no tests, no audit-grade structured
logging, and no pinned dependencies behind any of it.

CRITICAL FINDINGS
------------------------------------------------------------------------------
  PRAX-2026-06-15-001  LLM-produced shell commands are executed directly on
            the host via subprocess.run() with no approval gate, sandbox, or
            work-directory confinement.
      Action 1: Gate every command in `Runner.run_code()` behind an explicit
                human-approval step and an allow-list policy before calling
                `subprocess.run()`, and refuse commands that resolve outside
                the project work directory.
      Action 2: Implement the sandbox runner (`src/sandbox/firejail.py` /
                `code_runner.py`) so all generated code executes in an
                isolated environment rather than directly on the host.

  PRAX-2026-06-15-002  The declared sandbox modules firejail.py and
            code_runner.py are 0-byte empty files, so the promised execution
            isolation does not exist.
      Action: Implement `firejail.py` / `code_runner.py` to run generated code
              under OS-level isolation (firejail, container, or equivalent),
              or remove the sandbox claim from `ARCHITECTURE.md` and disable
              the `run` action until isolation exists.

  PRAX-2026-06-15-003  save_code_to_project() joins an LLM-controlled filename
            into the write path with no traversal check, allowing writes
            outside the project work directory.
      Action: In `save_code_to_project()`, resolve the joined path with
              `os.path.realpath` and reject any result that is not contained
              within the project work directory before calling
              `os.makedirs`/`open`.

  PRAX-2026-06-15-004  Untrusted web content and the user prompt reach code
            generation and execution with no injection screening, completing a
            web-to-RCE compound chain.
      Action 1: Add an injection-screening / trust-classification step on all
                web-retrieved content and the user prompt before they enter
                `formatter`/`coder`/`runner` context, and treat any
                "instruction-like" content as data.
      Action 2: Break the chain end-to-end by gating execution (PRAX-...-001)
                and implementing the sandbox (PRAX-...-002).

  PRAX-2026-06-15-005  The LLM-driven runner and decision agent can install
            packages and git-clone repositories with no user confirmation or
            registry verification.
      Action: Require explicit user confirmation before any install/clone
              command, restrict installs to a pre-approved package list, and
              reject unpinned or non-allow-listed registries.

  PRAX-2026-06-15-006  An unauthenticated POST /api/settings on 0.0.0.0:1337
            rewrites config.toml, letting any network-local caller alter
            endpoints and read/overwrite plaintext API keys.
      Action: Require authentication/authorization on all `/api/*` mutating
              routes, bind the server to loopback by default, and move
              provider credentials out of a world-writable plaintext config
              into a secret store.

BEHAVIOR SUMMARY
------------------------------------------------------------------------------
The dominant pattern is declared-but-absent enforcement: Sweep's remit makes
strong promises — screen all repository content for injection, never shell
out, never echo secrets — that the scoped code does not implement. Untrusted
issue, comment, file, and diff text flows into the LLM with no injection
screening, and the same LLM/repo-derived strings are interpolated into
subprocess.run(..., shell=True) at four search sites
(question_answerer.ripgrep, context_pruning.run_ripgrep_command,
dynamic_context_bot, client.is_file_bad), giving a content-to-shell path the
remit explicitly forbids. Separately, the GitHub webhook HMAC secret
(WEBHOOK_SECRET) is declared in config/server.py with a fail-open None default
and is never consulted by any in-scope code, and a PostHog project key is
hardcoded as a literal default in the same module.

CRITICAL FINDINGS
------------------------------------------------------------------------------
  PRAX-2026-06-15-001  GitHub webhook secret WEBHOOK_SECRET is declared with a
            fail-open None default and is never consulted by any in-scope
            code.
      Action 1: Fail closed on a missing webhook secret — assert
                WEBHOOK_SECRET is set at startup (like the existing
                OPENAI_API_KEY/COHERE_API_KEY asserts) so the agent never runs
                without HMAC verification configured.
      Action 2: Treat absent or invalid signatures as a hard reject in the
                request path, and add a regression test that a payload with
                no/invalid X-Hub-Signature-256 is refused.

  PRAX-2026-06-15-002  LLM-supplied ripgrep query is interpolated into
            subprocess.run with shell=True, giving a model-output-to-shell
            command-injection path.
      Action 1: Drop shell=True and pass rg arguments as a list
                (subprocess.run(["rg", ..., query], shell=False)) so the query
                is never parsed by a shell.
      Action 2: If a shell is unavoidable, shlex.quote() every model-derived
                argument before interpolation and add a test asserting
                metacharacters in `query` cannot break out.

  PRAX-2026-06-15-003  Two more shell=True ripgrep sites interpolate
            repo/model-derived search terms without escaping, despite an
            unused escape helper.
      Action 1: Convert both call sites to list-form subprocess.run(...,
                shell=False) so no shell parses the search term.
      Action 2: If shell form is retained, apply shlex.quote() (not the
                partial escape_ripgrep / manual quoting) to every interpolated
                term.

  PRAX-2026-06-15-004  No instruction-injection screening of issue, comment,
            file, or diff content before it enters the LLM context, as the
            remit mandates.
      Action 1: Add an injection-screening / quote-wrapping pass over issue,
                comment, file, and diff text at the boundary where it is added
                to model context, before any prompt assembly.
      Action 2: Label untrusted repository content distinctly in the prompt
                (delimited, marked external) and add adversarial test cases
                asserting injected instructions do not alter agent actions.
